toString()是从哪里来的

2.1 toString()是什么？

当查看各种文档时候，你会发现字符串有toString()，数组有toString()，时间对象也有toString()，其实任何东西都toString();但是有两样东西没有toString();

这是因为所有的对象都是继承自Object，而toString()是Object.prototype上的方法，也就是toString方法是Object的原型方法，而所有类型都是继承自Object，而你所创建的所有变量都是这些原始类型的实例，那js中的原始类型都是哪些？数值、字符串、布尔值
为什么这三种原始类型也是对象呢？因为这三种原始类型会在一定条件下自动转换为对象，也就是所谓的包装对象。

那什么又是包装对象呢？

  • 其实就是把上面三种原始类型包装成对应的Number、String、Boolean
    好了，那看过高程这么本js圣经的人都知道js的原始值类型有五种那两种null、和undefined是跑哪去了？为什么要null和undefined？他们不是都表示无吗？
    • 这个问题就要追溯js的设计的时候了。
      以下观点为我在阮老师的技术文档中看到的。

      js在设计之初只有null，表示“无”的值根据c语言的传统，null可以自动转为0
      但设计者觉得这样不好区分所以加入了undefined表示一个”无“的原始值，转换为数值是为NaN，但是null却很特殊，js把它包含在对象类型之中

那为什么alert会有[object Object]呢？

因为alert()自动调用了该参数的toString()方法。

那为什么第一个alert(fn)和第二个alert(fn)会有这么大不同呢？

第一个fn是函数，但是也是对象。函数是什么?是Function的实例。因为Function也是对象，也以也继承了Object的toString方法，但是同时也重写了此方法。所以才会出现第一种现象。

第二个fn是一个由constructor创建的对象，那它就是Object的实例，更是Fn的实例，但是Fn是什么？当使用new关键字的时候它是一个constructor，当不使用new的时候只是普通函数。这里使用了new，所以它是一个自定义对象类型。那自定义对象类型也是一个对象吧，它也是继承自Object的，但是这个Fn自定义类型重写了toString方法了吗？显然没有，所以此时fn调用的就是Object.prototype.toString方法。这个方法是具体是怎么定义的呢？查看mdn文档有以下一句解释。

The toString() method returns a string representing the object.

意思就是返回一个对象的字符串表示。

ecma规范要求toString 方法返回[object Class]

那第一个object是什么？是指对象。第二个大写的Object是指该对象类型。

因为各原始类型都重写了toString，所以可以显式调用Object.prototype.toString()来判断一个对象的类型。
