git命令的使用

发表于 2016-09-23   |   分类于 git   |  

  • git的工作流程———–要想学好git，下面这个图一定要记牢，要理解，不然你仍然茫然不知所措

  • Workspace :工作区

  • Index/Stage: 暂存区
  • Repository: 本地仓库区
  • Remote: 远程仓库
    在我把以下命令都一个个的实践之后，踩了不少的坑但是对上面这张图又有了新的认识
    所以我把这张图添加了一点东西，自己画了出来，字写的不好见谅

我的理解是：

工作区，暂存区，本地仓库都是只有一个
但是remote命令可以指定多个远程仓库，也就是一个本地仓库可以push到多个远程
仓库，而这里的remote是为本地仓库指定远程仓库的，
另外分支也是一样的，不同的分支下commit也是到不到的分支
另外吐嘈一下，谁说git很简单？

新建代码库

# 在当前目录新建一个Git代码库
# 同时这个命令会在本在文件夹新建一个.git文件夹，这个就是git用来管理这个仓库的版本的
$ git init

# 新建一个目录，将其初始化为Git代码库
$ git init [project-name]

# 下载一个项目和它的整个代码历史
# 这个url是从github上找
# git clone 这个命令一定要带参数，可选参数有很多
$ git clone [url]

配置

Git的设置文件为.gitconfig，它可以在用户主目录下（全局配置），也可以在项目目录下（项目配置）。

# 显示当前的Git配置
$ git config --list

# 编辑Git配置文件
$ git config -e [--global]

# 设置提交代码时的用户信息
$ git config [--global] user.name "[name]"
$ git config [--global] user.email "[email address]"

增加、删除文件

这里有必要说一下，我在提交过程中，假如你在一个文件中修改了a和b两处，a处你用add添加到了暂存区，但是b处你还没有添加到暂存区，如果此时commit的话，只会把add了a处的暂存区提交到本地仓库，而b处是不会commit的

# 添加指定文件到暂存区
$ git add [file1] [file2] ...

# 添加指定目录到暂存区，包括子目录
$ git add [dir]

# 添加当前目录的所有文件到暂存区
$ git add .

# 添加每个变化前，都会要求确认
# 这个-p 是指的patch，这个命令是坑，我还没实现
# 对于同一个文件的多处变化，可以实现分次提交
$ git add -p

# 删除工作区文件，并且将这次删除放入暂存区
$ git rm [file1] [file2] ...

# 停止追踪指定文件，但该文件会保留在工作区
$ git rm --cached [file]

# 改名文件，并且将这个改名放入暂存区
$ git mv [file-original] [file-renamed]

代码提交

  • 另外提一点这里的[]和-，一般前面跟一个-的是英文缩写，用两个–是英语全拼
  • 至于[]是指前面的选项的参数，你要到这个选项就必须要有参数
    git commit -m [message] 例如这个括里的message指的就是前面-m的参数，不过这也是我的理解，如有错误欢迎指正

分支


# 列出所有本地分支
$ git branch

# 列出所有远程分支
$ git branch -r

# 列出所有本地分支和远程分支
$ git branch -a

# 新建一个分支，但依然停留在当前分支
$ git branch [branch-name]

# 新建一个分支，并切换到该分支
$ git checkout -b [branch]

# 新建一个分支，指向指定commit
$ git branch [branch] [commit]

# 新建一个分支，与指定的远程分支建立追踪关系
$ git branch --track [branch] [remote-branch]

# 切换到指定分支，并更新工作区
$ git checkout [branch-name]

# 切换到上一个分支
$ git checkout -

# 建立追踪关系，在现有分支与指定的远程分支之间
$ git branch --set-upstream [branch] [remote-branch]

# 合并指定分支到当前分支
$ git merge [branch]

# 选择一个commit，合并进当前分支
$ git cherry-pick [commit]

# 删除分支
$ git branch -d [branch-name]

# 删除远程分支
$ git push origin --delete [branch-name]
$ git branch -dr [remote/branch]
